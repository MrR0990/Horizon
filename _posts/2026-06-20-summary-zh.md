## [imec、ASML、台积电实现 50nm 间距二维晶体管突破](https://www.tomshardware.com/tech-industry/semiconductors/imec-asml-and-tsmc-build-complementary-2d-material-transistors-at-50nm-pitch-on-a-300mm-wafer) ⭐️ 9.0/10

## [Project Valhalla 在 JDK 28 中引入值类型](https://www.jvm-weekly.com/p/project-valhalla-explained-how-a) ⭐️ 8.0/10

经过十年的开发，Project Valhalla 在 JDK 28 中为 JVM 引入了值类型，使得用户定义类型能够实现无对象头的紧凑内存布局。 这是 Java 的一次根本性演进，使开发者能够编写具有可预测内存布局的高性能代码，减少垃圾回收压力并提高缓存局部性。 值类型默认不能为 null，且没有对象标识，因此不能用于 synchronized 或 instanceof。JVM 将它们内联存储在数组和字段中，消除了每个元素的对象头。

hackernews · philonoist · 6月19日 06:35 · [社区讨论](https://news.ycombinator.com/item?id=48595511)

**背景**: 在传统 Java 中，每个对象都有一个对象头（在 64 位 JVM 上为 12-16 字节），用于存储标识哈希码和类指针等元数据。这种开销使得小对象的数组内存效率低下。Project Valhalla 引入了值类型（也称为内联类），它们像基本类型一样工作但由用户定义，结合了面向对象的抽象和基本类型的性能。

## [IETF 正式标准化 HTTP QUERY 方法，用于复杂只读请求](https://www.ithome.com/0/966/439.htm) ⭐️ 8.0/10

IETF 于 2026 年 6 月 16 日发布了 RFC 10008，将新的 HTTP QUERY 方法定义为提议标准。该方法允许携带请求体的安全、幂等且可缓存的只读查询，填补了 GET 和 POST 之间的空白。 该标准提供了一种语义正确的方式来执行复杂的只读查询，而无需滥用 POST，从而提高了 API 的清晰度和互操作性。它通过提供专用的查询方法，影响了 Web 开发者、API 设计者以及所有基于 HTTP 的服务。 QUERY 方法被定义为安全且幂等，支持通过 Accept-Query 头部等机制进行缓存，并可分配 URI 以缓存响应。目前它处于提议标准阶段，实际落地取决于服务器、代理和客户端的实现。

rss · IT之家 · 6月19日 15:16

**背景**: HTTP 方法如 GET 和 POST 具有不同的语义：GET 是安全且幂等的，但不能携带请求体；而 POST 可以携带请求体，但不保证安全。对于复杂的只读查询（例如带有大量或结构化参数的查询），开发者要么将数据塞入 GET 的 URL 中，要么滥用 POST，两者都有缺点。QUERY 方法通过结合 GET 的安全性和 POST 的携带请求体能力解决了这一问题。
