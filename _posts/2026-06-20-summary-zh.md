## [Project Valhalla 历经十年终抵 JDK 28](https://www.jvm-weekly.com/p/project-valhalla-explained-how-a) ⭐️ 8.0/10

Project Valhalla 在历经十年开发后，随 JDK 28 为 JVM 带来了值类型和堆扁平化功能。 这是一项重大的 JVM 增强，通过允许对象以内联方式存储（无需头部或间接引用）来提升性能和内存效率，惠及 Java 及其他 JVM 语言。 值类型是没有标识的引用类型，意味着它们不能为 null 且按值比较；堆扁平化去除了对象头部和指针，使得数组中可以密集存储。

hackernews · philonoist · 6月19日 06:35 · [社区讨论](https://news.ycombinator.com/item?id=48595511)

**背景**: Project Valhalla 是一项长期努力，旨在为 JVM 添加值类型——类似于基本类型但拥有方法和字段。值类型没有标识且不可变，非常适合高性能数据结构。堆扁平化将值对象内联存储在数组和字段中，减少内存开销并改善缓存局部性。

## [IETF 发布 HTTP QUERY 新方法标准 RFC 10008](https://www.ithome.com/0/966/439.htm) ⭐️ 8.0/10

互联网工程任务组（IETF）发布了 RFC 10008，将新的 HTTP QUERY 方法定义为提议标准。该方法允许携带请求体的复杂只读请求，填补了 GET 和 POST 之间的空白。 该标准提供了一种语义正确的方式来执行复杂查询，而无需滥用 POST，提高了 API 设计的清晰度，并支持安全、幂等的只读操作。它将影响 Web 开发者、API 设计者以及所有基于 HTTP 的服务。 QUERY 被定义为安全且幂等，类似于 GET，但允许像 POST 一样携带请求体。该规范还引入了 Accept-Query 响应头，并支持缓存，可为后续 GET 请求分配 URI。

rss · IT之家 · 6月19日 15:16

**背景**: HTTP 方法如 GET 和 POST 具有不同的语义：GET 用于安全、幂等的数据检索，但存在 URL 长度限制；而 POST 可以携带请求体，但不保证是只读的。开发者经常使用 POST 进行复杂查询，这在语义上是不正确的。QUERY 方法通过结合 GET 的安全性和 POST 的携带请求体能力解决了这一问题。
